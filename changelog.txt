v0.17.0
- Books and podcast episodes in library listings now show a
  "[NN%]" prefix when in progress — same format Continue Listening
  uses, so an item looks the same whether you reach it via resume
  or by browsing. Covers All Books, Series, Authors, Collections,
  Podcast Episodes, and Recent Episodes.

v0.16.0
- Resume no longer leaks a fraction of a second from the start of the
  book. On resume, Koshelf sets inputstream.tempo.start_time in addition
  to the usual audiobook_bookmark property. inputstream.tempo 0.3.6+
  uses that hint to hold packet output until PAPlayer's bookmark seek
  arrives (or a 2 s safety timeout) — so no pts=0 audio reaches the
  sink before the seek lands. Requires inputstream.tempo 0.3.6.

v0.15.0
- Library listings now sort server-side via ABS, so the order is
  consistent across pages instead of only within the loaded page.
  A "[ Sort: ... ]" item at the top opens a picker (Title, Author,
  Recently added/updated, Duration, Published year, Random). The
  choice is preserved through pagination.
- Background art: listing items expose their cover as fanart so
  skins with a blurred-fanart visualization (Estuary and similar)
  use the book/podcast cover as a backdrop during playback.
- Info dialog: books in Continue Listening now include the book
  description, and every playable item reports mediaType='song' so
  most skins show the full music Info dialog with description.

v0.14.0
- Removed the Player setting and the Video player path. PAPlayer is now
  the only player. The video path was originally added to chase a
  tempo-tracking OSD, but its state.time progressed at wall-clock rather
  than tempo-adjusted rate, so it offered no benefit over the music
  player path.

v0.13.0
- New Player setting: choose between the Music player (PAPlayer, default)
  and the Video player (VideoPlayer). Tempo, resume, position sync, and
  per-book speed tracking all work in both modes. The video player path
  uses VideoPlayer's StartOffset for resume plus a service-side
  onAVStarted fallback. The addon now declares <provides>audio video</provides>.
- Requires inputstream.tempo 0.3.1 or newer so the OSD position tracks
  content time at non-1× tempo.

v0.12.3
- Resume playback uses PAPlayer's native audiobook_bookmark property.
  PAPlayer seeks before audio output begins — no race conditions, no
  Python seek hack, no C++ intercept.
- Rename KoShelf → Koshelf throughout.
- Configurable items per page (25–500, default 100) for full client-side
  sorting of large libraries.
- Episode title shown on OSD (uses session.displayTitle); album set to
  podcast name for episodes.
- Description available in info dialog (press i) via setComment() in all
  browsing routes and during playback.
- Now playing root label: speed at front, always two decimal places.
- Max speed lower bound changed from 1.5x → 1.0x (allows disabling tempo).
- Speed step description updated to reference PgUp/PgDn keys.
- Sync guard prevents overwriting a valid resume position with 0.

v0.10.5
- Service-triggered root refresh when tempo sentinel appears/disappears.
- Move Playback section to first in settings.
- Tighten service loop to 0.25s for faster resume-seek latency.

v0.10.4
- Continue Listening default sort uses SORT_METHOD_NONE (server returns
  items in last-played-descending order) and exposes LastPlayed, Title,
  Artist, Duration as alternatives.
- Progress percentage moved into the InfoTag title so it survives album-
  style views that only show the tag title.
- Add artist field to podcast items.
- Cache root directory is invalidated (cacheToDisc=False) so the "Now
  playing" row reappears on next root visit.

v0.10.3
- Drop setDateAdded from InfoTagMusic (it doesn't exist — only VideoTag has
  one) and SORT_METHOD_DATEADDED from book sorts. Was crashing every
  listing after the v0.10.0 sort/date additions.

v0.10.2
- Show a "Now playing: <title> — <speed>" item at the root when tempo is
  the active inputstream; selecting it opens a speed picker.

v0.10.1
- Fix settings rendering for Kodi 21: sliders use <format>{:.2f}x</format>
  inline instead of the formatlabel-with-id pattern, speed_step uses
  type=string with option-label string IDs, and empty-default fields
  declare <allowempty>true</allowempty>.

v0.10.0
- Add sort-order options to every browsing view (title, artist, album,
  duration for books; last played/title/artist/duration for Continue
  Listening; date/title/duration for podcast episodes).
- Continue Listening defaults to last-played descending.
- Drop API key auth; username + password only (cached session token
  handled transparently).
- Bind Up/Down to BigStepForward/BigStepBack in audio contexts for
  progressive skip. (Moved out to inputstream.tempo's keymap in later
  versions.)

v0.9.1
- Migrate settings.xml to Kodi 21's structured format with help text,
  slider controls for speed ranges, and a Playback/General section split.
- Speed step (spinner) + min/max/book/podcast speed sliders formatted
  as "1.50x".
- Clamp saved per-book speeds against the current min/max at replay.
- Share step/min/max with inputstream.tempo via a JSON file in
  special://temp/inputstream_tempo_config.
- Drop the speed_mode setting — always step by speed_step now.
- Skip ebook-only library items in both Continue Listening and regular
  library listings.
